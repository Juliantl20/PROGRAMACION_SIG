1 Ejercicio 1: Hidrología del Río Magdalena

1.1 Contexto

Se analiza el comportamiento de cinco estaciones hidrológicas del IDEAM ubicadas a lo largo del río Magdalena. Para cada estación se cuenta con el caudal medio mensual expresado en metros cúbicos por segundo, es decir, en m³/s.

Las estaciones consideradas son:

  • Honda
  • Puerto Berrío
  • Barrancabermeja
  • Puerto Wilches
  • Calamar

El objetivo es calcular estadísticos básicos y transformar los caudales de m³/s a litros por segundo, l/s, usando una operación vectorizada.

1.2 Desarrollo en R

# ============================================================
# EJERCICIO 1: HIDROLOGÍA DEL RÍO MAGDALENA
# ============================================================

# Definir una colección ordenada con los nombres de las estaciones
estaciones <- c(
  "Honda",
  "Puerto Berrío",
  "Barrancabermeja",
  "Puerto Wilches",
  "Calamar"
)

# Definir una colección paralela con los caudales en m3/s
caudales_m3s <- c(1500, 2100, 2800, 3200, 4500)

# Calcular el caudal máximo registrado
caudal_maximo <- max(caudales_m3s)

# Calcular el caudal promedio de las cinco estaciones
caudal_promedio <- mean(caudales_m3s)

# Convertir los caudales de m3/s a litros por segundo
# 1 m3 = 1000 litros
# Esta operación se realiza de forma vectorizada, sin usar bucles for.
caudales_ls <- caudales_m3s * 1000

# Imprimir resultados
estaciones
[1] "Honda"           "Puerto Berrío"   "Barrancabermeja" "Puerto Wilches" 
[5] "Calamar"        
caudales_m3s
[1] 1500 2100 2800 3200 4500
caudal_maximo
[1] 4500
caudal_promedio
[1] 2820
caudales_ls
[1] 1500000 2100000 2800000 3200000 4500000

El caudal máximo registrado es de 4500 m³/s. El caudal promedio de las cinco estaciones analizadas es de 2820 m³/s.

La transformación de m³/s a l/s se hizo multiplicando el vector caudales_m3s por 1000. En R, esta operación se aplica automáticamente a cada elemento del vector, por lo cual no fue necesario usar un ciclo for.

3 Preguntas analíticas

3.1 Pregunta Ejercicio 1: Vectorización

Si el Ejercicio 1 se desarrollara en Python y se creara caudales_m3s como una lista nativa, por ejemplo:

caudales_m3s = [1500, 2100, 2800, 3200, 4500]
caudales_m3s * 1000

la operación no realizaría la multiplicación elemento por elemento. En Python, cuando una lista nativa se multiplica por un número entero, la lista se repite esa cantidad de veces. Es decir, el resultado sería una lista mucho más larga con los mismos valores repetidos, no la conversión de m³/s a l/s.

Por ejemplo, una versión reducida de ese comportamiento sería:

[1500, 2100] * 3

El resultado sería:

[1500, 2100, 1500, 2100, 1500, 2100]

Esto demuestra que las listas nativas de Python no se comportan como vectores para este tipo de operación.

La librería de Python que soluciona este problema es NumPy. Al convertir la lista en un arreglo numérico de NumPy, la multiplicación sí se aplica elemento por elemento:

import numpy as np

caudales_m3s = np.array([1500, 2100, 2800, 3200, 4500])
caudales_ls = caudales_m3s * 1000

En ese caso, el resultado sería:

array([1500000, 2100000, 2800000, 3200000, 4500000])

La diferencia con R, es que R trabaja de forma natural con vectores numéricos. Por eso, en R la operación:

caudales_m3s * 1000
[1] 1500000 2100000 2800000 3200000 4500000

multiplica automáticamente cada elemento del vector por 1000.

En Julia también puede realizarse una operación elemento a elemento, aunque suele expresarse de forma explícita mediante el operador punto:

caudales_ls = caudales_m3s .* 1000

En conclusión, Python con listas nativas no realiza la operación matemática deseada, sino una repetición de la lista. Para obtener un comportamiento vectorizado se debe usar una estructura especializada como un arreglo de NumPy. En cambio, R permite la vectorización directamente sobre sus vectores numéricos.

3.2 Pregunta General 2: Indexación

En el lenguaje elegido para este trabajo, R, la línea exacta para extraer las tres primeras estaciones del DataFrame del Ejercicio 2 usando indexación por rangos es:

df_aire[1:3, ]
  estacion pm25
1 Carvajal   55
2  Kennedy   42
3 Fontibón   38

Esta instrucción selecciona las filas 1, 2 y 3 del DataFrame df_aire, conservando todas las columnas. El resultado es:

df_aire[1:3, ]
  estacion pm25
1 Carvajal   55
2  Kennedy   42
3 Fontibón   38

R utiliza indexación Base-1, lo que significa que la primera posición de una colección se identifica con el número 1. Además, en R una secuencia como 1:3 incluye tanto el límite inferior como el límite superior. Por esa razón, df_aire[1:3, ] devuelve exactamente las tres primeras estaciones: Carvajal, Kennedy y Fontibón.

En contraste, Python utiliza indexación Base-0 y el límite superior del rango se excluye. En pandas, una instrucción equivalente sería:

df_aire.iloc[0:3]

Aunque se escribe 0:3, Python devuelve las posiciones 0, 1 y 2, pero excluye la posición 3. Por tanto, mientras R incluye el límite superior en una expresión como 1:3, Python excluye el límite superior en una expresión como 0:3.
